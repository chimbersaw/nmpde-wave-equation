\section{Introduction}

The wave equation is a second-order hyperbolic partial
differential equation.  In two space dimensions it models, for example, the
transverse displacement of a membrane, acoustic pressure waves, or idealized
elastic vibrations.  Its numerical approximation is less forgiving than the
approximation of parabolic problems: besides consistency and stability, one
must also control non-physical dissipation and dispersion.  Numerical
dissipation changes the amplitude of the wave, while numerical dispersion
changes its phase velocity and gradually shifts the wave out of phase with the
exact solution.

The project goal was to build a finite element solver for
\begin{equation}
\begin{cases}
\dfrac{\partial^2 u}{\partial t^2} - c^2 \Delta u = f, & \text{in } \Omega \times (0,T],\\
u = g, & \text{on } \partial\Omega \times [0,T],\\
u(\cdot,0)=u_0,\qquad \dfrac{\partial u}{\partial t}(\cdot,0)=u_1, & \text{in } \Omega,
\end{cases}
\label{eq:strong}
\end{equation}
where $\Omega\subset\R^2$ is a bounded domain and $c>0$ is the wave speed.  The
implemented code supports several domains and boundary data through
configuration files: standing waves on the square $[0,5]^2$, Gaussian pulse
propagation, driven boundaries, a circle, a rectangular domain and an donut.
The numerical experiments focus on the square domain because it admits a simple
exact solution and therefore allows a clean convergence and energy analysis.

The solver is written in C++ with deal.II and distributed linear algebra through
Trilinos/MPI.  Meshes are generated by Gmsh and stored in the \texttt{mesh/}
directory. Experiment definitions are stored in \texttt{configs/}.  The
implementation writes ParaView-compatible \texttt{.vtu}/\texttt{.pvtu} files and
CSV diagnostics used to produce the plots in \texttt{results/}.

\section{Continuous Problem}

\subsection{Weak Formulation}

For the weak formulation we use the Sobolev space
\[
H^1(\Omega)
=\left\{v\in L^2(\Omega): \frac{\partial v}{\partial x_i}\in L^2(\Omega),
\ i=1,2\right\},
\]
where the derivatives are understood in the distributional (weak) sense.  The space
$H_0^1(\Omega)$ is the subspace of $H^1(\Omega)$ with zero trace on the
boundary:
\[
H_0^1(\Omega)=\{v\in H^1(\Omega): v=0 \text{ on } \partial\Omega\}.
\]
For homogeneous Dirichlet data, the natural test space is therefore
\[
V = H_0^1(\Omega).
\]
Multiplying \eqref{eq:strong} by $v\in V$, integrating over $\Omega$, and using
Green's formula gives
\begin{equation}
\int_\Omega u_{tt}\,v\,\dd x
+ c^2\int_\Omega \nabla u\cdot\nabla v\,\dd x
= \int_\Omega f v\,\dd x
\qquad \forall v\in V.
\label{eq:weak}
\end{equation}
The boundary integral vanishes because $v=0$ on $\partial\Omega$.  If
non-homogeneous Dirichlet data are prescribed, one may introduce a lifting
$R_g(\cdot,t)$ such that $R_g=g$ on the boundary and write
$u=\widehat u+R_g$ with $\widehat u\in H_0^1(\Omega)$.  The same weak equation
then holds for the homogeneous unknown $\widehat u$, with additional known
terms on the right-hand side.  In the code this idea is enforced algebraically:
boundary values are interpolated at the boundary degrees of freedom and imposed
strongly in the linear systems.  This is the standard Cauchy-Dirichlet weak
formulation for the wave equation. Salsa writes the same weak form using
$\ddot u$ as a notation for $u_{tt}$ \cite{salsa}.

The energy of the unforced homogeneous problem is
\begin{equation}
E(t)=\frac12\int_\Omega |u_t|^2\,\dd x
    +\frac{c^2}{2}\int_\Omega |\nabla u|^2\,\dd x .
\label{eq:continuous-energy}
\end{equation}
For $f=0$ and fixed homogeneous boundary values, differentiating
\eqref{eq:continuous-energy} and integrating by parts shows $E'(t)=0$, this
energy conservation property is one of the main qualitative features that a
good wave solver should preserve or approximate well.  This energy calculation
is also the basis of the classical a priori estimates for the Cauchy-Dirichlet
wave problem \cite{salsa}.

The balance is also useful for interpreting the driven-boundary examples.  If
the boundary is not fixed, the same calculation gives
\begin{equation}
\frac{\dd E}{\dd t}
=\int_\Omega f\,u_t\,\dd x
 + c^2\int_{\partial\Omega}\partial_n u\,u_t\,\dd s .
\label{eq:energy-flux}
\end{equation}
The volume term is the work of the source and the boundary term is the energy
flux through the boundary.  Therefore an increase of the energy in the driven
left-boundary or two-sided-boundary videos is expected if the prescribed boundary displacement performs work on the
membrane.  Conversely, in the homogeneous standing-wave diagnostics, the correct
reference behavior is constant energy, deviations from this reference isolate
algorithmic dissipation or instability.

\subsection{Manufactured Standing Wave}

The main verification test uses the exact solution
\begin{equation}
u_{\mathrm{ex}}(x,y,t)
= \sin\!\left(\frac{\pi x}{5}\right)
  \sin\!\left(\frac{\pi y}{5}\right)
  \cos\!\left(\frac{c\pi\sqrt2}{5}t\right)
\label{eq:standing-wave}
\end{equation}
on $\Omega=[0,5]^2$.  This function vanishes on the boundary, so the boundary
condition is homogeneous.  The corresponding initial data and forcing are
implemented in \texttt{function\_factory.cpp}.  The forcing is computed as
$f=(u_{\mathrm{ex}})_{tt}-c^2\Delta u_{\mathrm{ex}}$, which makes the
analytical solution exactly compatible with \eqref{eq:strong}.

\section{Spatial Discretization}

Let $\mathcal{T}_h$ be a triangular mesh of $\Omega$ and let
$V_h\subset H^1(\Omega)$ be the continuous finite element space generated by
deal.II's \texttt{FE\_SimplexP}.  In the performed experiments the finite
element degree is $r=1$, hence the expected best asymptotic behavior is
$O(h^2)$ in the $L^2$ norm and $O(h)$ in the $H^1$ norm when the temporal error
is sufficiently small.

Using the basis $\{\varphi_i\}_{i=1}^{N_h}$ and writing
\[
u_h(x,t)=\sum_{j=1}^{N_h} U_j(t)\varphi_j(x),
\]
the semi-discrete problem takes the matrix form
\begin{equation}
\M \ddot{\Uh}(t) + c^2\K\Uh(t)=\F(t),
\label{eq:semidiscrete}
\end{equation}
with
\begin{equation}
M_{ij}=\int_\Omega \varphi_j\varphi_i\,\dd x,
\qquad
K_{ij}=\int_\Omega \nabla\varphi_j\cdot\nabla\varphi_i\,\dd x,
\qquad
F_i(t)=\int_\Omega f(t)\varphi_i\,\dd x .
\end{equation}
In the implementation the stiffness matrix stores only the geometric bilinear
form $\int\nabla\varphi_j\cdot\nabla\varphi_i$, the factor $c^2$ is inserted in
the time-stepping systems and in the energy calculation.  This keeps the
assembled matrices reusable when the wave speed changes.

The matrices are assembled cell by cell using Gaussian quadrature of order
\texttt{fe\_degree + 1}.  The triangulation is first read from a Gmsh
\texttt{.msh} file, then partitioned across MPI ranks and converted into a
deal.II fully distributed triangulation.  Locally owned and locally relevant
degree-of-freedom sets are stored separately, which is essential for point
evaluation, output and interface-cell computations in a distributed mesh.

\section{Time Discretization}

Two time discretization families are implemented.  The $\theta$-method is
applied after rewriting the second-order equation as a first-order system,
whereas the Newmark method acts directly on \eqref{eq:semidiscrete}.

\subsection{The $\theta$-Method}

Introduce $\Vh=\dot{\Uh}$.  The first-order system is
\begin{equation}
\dot{\Uh}=\Vh,\qquad
\M\dot{\Vh}+c^2\K\Uh=\F.
\end{equation}
The $\theta$-method updates $\Uh^n,\Vh^n$ to $\Uh^{n+1},\Vh^{n+1}$ by
\begin{align}
\Uh^{n+1} &= \Uh^n+\dt\left((1-\theta)\Vh^n+\theta\Vh^{n+1}\right),
\label{eq:theta-u}\\
\M\frac{\Vh^{n+1}-\Vh^n}{\dt}
&+c^2\K\left((1-\theta)\Uh^n+\theta\Uh^{n+1}\right)
=(1-\theta)\F^n+\theta\F^{n+1}.
\label{eq:theta-v}
\end{align}
Substitution of \eqref{eq:theta-u} into \eqref{eq:theta-v} produces the system
actually used in the code:
\begin{equation}
\left(\M+c^2\theta^2\dt^2\K\right)\Vh^{n+1}=R_\theta^n,
\label{eq:theta-system}
\end{equation}
where $R_\theta^n$ contains only known quantities.  Once the velocity is known,
\eqref{eq:theta-u} gives the displacement.  The implemented choices are:
\[
\theta=0 \quad\text{Forward Euler},\qquad
\theta=\frac12 \quad\text{Crank-Nicolson},\qquad
\theta=1 \quad\text{Backward Euler}.
\]
Forward Euler is explicit in the stiffness part, but it is not stable for the
oscillatory wave dynamics considered in this report.
Crank-Nicolson is second-order accurate and energy-conserving for the linear
homogeneous problem.  Backward Euler is first-order accurate but strongly
dissipative. It is robust but not well suited to long-time wave propagation if
the physical amplitude is important.

\subsection{Newmark Methods}

The Newmark family advances displacement, velocity and acceleration:
\begin{align}
\Uh^{n+1}
&=\Uh^n+\dt\Vh^n+\dt^2\left(\left(\frac12-\beta\right)\Ah^n+\beta\Ah^{n+1}\right),
\label{eq:newmark-u}\\
\Vh^{n+1}
&=\Vh^n+\dt\left((1-\gamma)\Ah^n+\gamma\Ah^{n+1}\right).
\label{eq:newmark-v}
\end{align}
The new acceleration satisfies
\[
\M\Ah^{n+1}+c^2\K\Uh^{n+1}=\F^{n+1}.
\]
For $\beta>0$, substituting \eqref{eq:newmark-u} gives a linear system for the
new displacement:
\begin{equation}
\left(\frac{1}{\beta\dt^2}\M+c^2\K\right)\Uh^{n+1}=R_N^n.
\label{eq:newmark-system}
\end{equation}
The acceleration and velocity are then recovered algebraically.  The two
implemented variants are:
\[
\beta=0,\ \gamma=\frac12 \quad\text{leapfrog/central difference},\qquad
\beta=\frac14,\ \gamma=\frac12 \quad\text{avg accel}.
\]
The leapfrog case is conditionally stable and very sensitive to the restriction
$\dt \le Ch$.  The avg accel method is unconditionally stable for the linear
model and, as confirmed by the diagnostics, conserves the discrete energy up to
solver tolerance.

For the linear undamped second-order system considered here, Crank-Nicolson
applied to the first-order system $(\Uh,\Vh)$ is algebraically equivalent to the
Newmark avg accel method with $\beta=1/4$ and $\gamma=1/2$.  Both are the
trapezoidal rule written in different variables.  The code implements them
through different linear systems, so small differences could appear from
boundary handling or solver tolerances, but the mathematical time discretization
is the same for the homogeneous standing-wave tests.

The stability limitation of explicit schemes is not only a formal statement.
The numerical domain of dependence must be wide enough to contain the physical
domain of dependence.  If the step is too large compared with the mesh size and
the wave speed, high-frequency components are amplified and the energy grows
rapidly.  This is why the same spatial discretization may look accurate for a
small enough time step and completely fail for a larger one.  Implicit schemes
avoid this blow-up, but they still differ qualitatively: Backward Euler
suppresses high frequencies by damping, while Crank-Nicolson and Newmark avg
accel keep the energy but can shift the phase of the wave when $\dt$ is large.
This distinction between amplification error and dispersion error is exactly
the type of analysis used by Quarteroni for finite element schemes for
hyperbolic equations \cite{quarteroni}.

\section{Implementation}

\subsection{Code Structure}

The executable is configured entirely through text files.  Each run specifies
the mode, mesh, finite element degree, wave speed, time step, number of steps,
initial data, forcing, boundary data and time integration parameters.  Relative
paths are resolved with respect to the directory of the configuration file,
which makes the examples reproducible from any working directory.

The main classes are:
\begin{itemize}
\item \texttt{WaveProblemConfig}, which parses and validates the configuration;
\item \texttt{WaveSolver}, which owns the mesh, DoFHandler, matrices, vectors,
      boundary enforcement, output and diagnostics;
\item \texttt{ThetaTimeIntegrator} and \texttt{NewmarkTimeIntegrator}, which
      implement the two time stepping families;
\item \texttt{ConvergenceRunner}, which repeats solves over mesh or time-step
      lists and writes CSV files with errors and observed orders;
\item \texttt{function\_factory.cpp}, which maps names such as
      \texttt{standing\_wave\_5x5}, \texttt{gaussian\_pulse} or
      \texttt{two\_sided\_driven\_boundary} to deal.II functions.
\end{itemize}

The same factory also provides the initial data and forcing terms used by the
experiments: the manufactured standing wave on $[0,5]^2$, its velocity and
forcing, the circular standing-wave data, a Gaussian pulse centered at
$(2.5,2.5)$, and the annular angular mode.

The code uses conjugate gradients with a Trilinos SSOR preconditioner for the
SPD systems.  Boundary values are imposed by \texttt{MatrixTools::apply\_boundary\_values}
in solve systems, and by direct insertion when only a vector needs to be made
consistent.  For time-dependent Dirichlet data, boundary velocities and
accelerations are approximated by finite differences of the prescribed boundary
function.  This is important in the driven examples, where the boundary itself
injects energy into the computational domain.

\subsection{Diagnostics and Output}

The solver writes solution files for ParaView and can also write per-step
diagnostics.  The discrete energy used in the diagnostics is
\begin{equation}
E_h^n
=\frac12(\Vh^n)^T\M\Vh^n
 +\frac{c^2}{2}(\Uh^n)^T\K\Uh^n .
\label{eq:discrete-energy}
\end{equation}
For the standing wave diagnostic, a probe at $(2.5,2.5)$ is also logged and
compared against the exact trace.  This makes dissipation and dispersion easy to
separate: amplitude loss appears directly in the energy and in the probe trace,
whereas phase error appears as a time shift between the numerical and exact
signals.

The repository also contains a \texttt{videos/} directory with ParaView
visualizations for representative configurations. In particular, the
videos show that Crank-Nicolson and Newmark avg accel give the
cleanest and most stable wave propagation, while Leapfrog and Backward Euler
show more visible numerical artifacts.  Forward Euler is unstable when the time
step is not compatible with the explicit stability restriction. The \texttt{pulse}
subfolder contains runs at different time steps that make this instability
visible.

\section{Convergence Study}

The convergence study uses the exact standing wave \eqref{eq:standing-wave}.
Two separate sweeps are performed.  The spatial sweep keeps the time step small
and refines the mesh from \texttt{h0p5} to \texttt{h0p0625}.  The temporal sweep
uses a fixed fine mesh and varies $\dt$.  The final time is $T=1$ for most
schemes, except for Newmark avg accel where the configured run uses
$T=5$.

\[
\|u-u_h\|_{L^2(\Omega)}=O(h^{r+1}),\qquad
\|u-u_h\|_{H^1(\Omega)}=O(h^r),
\qquad r=1.
\]

time convergence, as predicted by the method.  The explicit schemes require
more care: the plots are useful precisely because they show the stability
restriction.  Once the time step violates the admissible stability range, the error
does not represent convergence anymore, it represents numerical divergence.

\section{Dissipation and Dispersion}

The dissipation and dispersion audit runs all five schemes on the same standing
wave problem up to $T=5$, with time steps
\[
\dt\in\{0.15, 0.1, 0.05, 0.02, 0.01, 0.005, 0.001\}.
\]
The mesh is \texttt{square\_5x5\_h0p25.msh}, the wave speed is $c=4$, and the
probe point is the center of the square.  Since
$\sin(\pi x/5)\sin(\pi y/5)=1$ at $(2.5,2.5)$, the exact probe is simply a
cosine in time.

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{dissipation_energy.png}
\caption{Normalized discrete energy $E_h(t)/E_h(0)$.  Crank-Nicolson and
Newmark avg accel preserve energy almost exactly.  Backward Euler
damps the solution, with stronger damping for larger $\dt$.  Forward Euler and
Newmark leapfrog diverge when the time step is outside the stable range.}
\label{fig:dissipation}
\end{figure}

Figure~\ref{fig:dissipation} is the clearest numerical confirmation of the
theory.  Crank-Nicolson and avg accel lie on the reference energy
line for all tested time steps.  Backward Euler is unconditionally stable but
very dissipative: for $\dt=0.15$ the final energy ratio is approximately
$2.54\cdot10^{-4}$, and even for $\dt=0.01$ only about $53\%$ of the initial
energy remains at $T=5$.  Forward Euler diverges for all time steps in this
diagnostic sweep, while Newmark leapfrog is stable for $\dt\le0.02$ and
diverges for the larger steps.

\;\\

The probe traces in Figure~\ref{fig:dispersion} separate amplitude and phase
effects.  Backward Euler visibly damps the oscillation and also shifts the
phase.  Crank-Nicolson and Newmark avg accel keep the amplitude
almost perfectly, but large time steps introduce a visible phase lag.  This is
the typical behavior of non-dissipative second-order schemes for wave problems:
the energy is preserved, yet the numerical wave speed is not exact.  Leapfrog
matches the exact trace well in the stable small-step cases, but its stability
margin is narrow.  Forward Euler is qualitatively unsuitable here because the
energy grows instead of remaining bounded.

\section{Limitations and Possible Improvements}

The project already covers the requested numerical aspects, but several
limitations are visible from the results.  First, the explicit methods are not
protected by an automatic explicit-stability estimator in the current configuration files.
This is why the Forward Euler and leapfrog sweeps contain divergent points.  A
practical improvement would be to compute a conservative estimate of
$h_{\min}/c$ after mesh loading and reject explicit runs whose time step is too
large.  The report intentionally keeps the unstable data because it documents
the actual behavior of the solver, but an automated guard would make production
runs safer.

Second, the explicit methods still solve systems involving the mass matrix.
This is acceptable for a unified implementation and for the project scale, but
it is not the most efficient version of an explicit wave solver.  A mass-lumped
formulation would replace these solves by diagonal scaling and would make
Forward Euler or leapfrog much cheaper per time step.  The trade-off is that
mass lumping changes the spatial discretization and may affect dispersion, so it
should be evaluated with the same energy and probe diagnostics used in this
work.

Third, the scalability study is limited to one method and one problem size.  It
is still useful because it confirms that the distributed data structures work,
but it does not fully characterize the solver.  A more complete performance
study would repeat the strong-scaling experiment for all schemes, include weak
scaling, and separate setup, assembly, solve and output times.  Such a split
would make it easier to distinguish communication overhead from linear solver
cost and from file I/O.

Finally, all reported convergence runs use linear finite elements.  The code
allows \texttt{fe\_degree > 1}, so a natural extension would be to repeat the
study with quadratic elements.  This would test the expected $O(h^3)$ $L^2$
behavior and would also make dispersion effects smaller for the same mesh size.

\section{Conclusions}

The implemented solver follows the standard finite element semi-discretization
of the wave equation and provides two families of time integration schemes.  The
numerical results are consistent with the main theoretical expectations.  Linear
finite elements give approximately second-order convergence in $L^2$ and
first-order convergence in $H^1$ when the time discretization is stable and the
temporal error is not dominant.  Backward Euler is first-order in time and
strongly dissipative.  Crank-Nicolson and Newmark avg accel are the
best qualitative choices among the tested methods because they preserve the
discrete energy and remain stable over long simulations, their main remaining
error is dispersion for large time steps.  Leapfrog can be accurate and
energy-preserving in the stable regime but is limited by the restriction $\dt\le Ch$.
Forward Euler is the least robust method and shows instability in the diagnostic
runs.

From an implementation perspective, the project supports reproducible
configuration-driven runs, external Gmsh meshes, distributed deal.II
triangulations, strong Dirichlet boundary enforcement, ParaView output,
convergence sweeps and energy diagnostics.  The videos
and plots agree: Newmark avg accel and
Crank-Nicolson give the most stable visual results, while the dissipative or
explicit schemes show the expected artifacts.  Overall, Newmark avg accel
offers the best balance in this codebase: it is stable,
energy-preserving, visually clean and supported by the strongest convergence
and diagnostic evidence.
